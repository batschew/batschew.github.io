    • Project: Installing Linux to hardware

    Outline

    For this project, I decided to install a Linux operating system distribution to a physical computer. I have set up Linux virtual machines in the past, but prior to this project, I had not installed it to hardware.

    The distribution that I chose, Pop!_OS, is derived from the popular Ubuntu distribution. It is maintained by a US-based hardware company called System76.

    Installing Linux to hardware is simple if you plan to overwrite the contents of the disk, but I wanted to preserve Windows 10 on a separate partition so I could boot into either system at my choosing. This setup, termed dual-booting, is significantly more complex.


    Process

    After burning an ISO of Pop!_OS to a flash drive and booting into the installer, I encountered errors with the graphical display. I inferred that these were due to the fact that the hardware I was installing Pop!_OS to was using an NVIDIA graphics card with proprietary drivers. As Linux is a free and open source operating system, it usually does not provide these drivers out of the box.

    I was able to resolve this problem by entering a text-only command line, from which I was able to download the drivers from the distribution's repositories. After this was completed, the graphical errors were resolved and I was able to resume the installation process.

    I manually partitioned the disks (an SSD and an HDD) to provide space for the distribution, while preserving the data on the Windows partitions.
